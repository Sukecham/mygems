本資料の目的

本日確認したいのは、最終的な文章表現や細部の設計そのものではありません。
資金制約・期間制約・病院側の条件・採択可能性を踏まえたうえで、今回の実証をどのような構造の事業として位置づけるか——その全体方向性について合意を形成することが主旨です。方向性の合意が取れた後、提案書への文章化・落とし込みは当方で進めます。

01

提案の基本思想

今回の事業を単なる生成AIツールの導入で終わらせないことが、提案全体の出発点です。現場調査に始まり、業務棚卸し、課題の整理、優先順位付け、再構築案の提示、そして病院の意思決定に基づく改善実行へとつなげる——すなわち本質的な医療DX・AXの実証事業として構成することを基本方針とします。

```
現場調査
→
業務棚卸し
→
課題整理
→
優先順位付け
→
再構築案
→
改善実行

さらに、改善を単年度の外部支援で終わらせるのではなく、並行して職員やコアメンバーに教育パッケージを提供し、生成AIやデジタル技術を活用しながら自施設内で改善を継続できる範囲——内製化可能領域を広げていくことを重要な柱として据えます。

```
02

生成AI利用基盤の二層構造

医療情報を全く扱わない形では、スマートアイランド事業としてのインパクトが弱くなる懸念があります。そのため、電子カルテ系情報と安全に接続・管理できる生成AI利用環境は、提案に必ず含める必要があると考えます。

```
制約条件の認識

ただし、時間的制約・資金的制約を踏まえ、電子カルテベンダーとの大きな契約変更や保守契約の見直し、仕様変更を前提とするような重い実装は極力回避すべきです。ベンダーへの確認・調整は必要になり得るとしても、契約や仕様の大幅な変更が前提となる構成はリスクが高すぎると判断しています。

この制約を踏まえた現実的かつ説得力のある構成として、生成AI利用基盤を二層で整備する方向性を提案します。

Layer 1 — 医療情報系
医療情報対応 生成AI環境
電子カルテ系情報との接点を持ち得る、医療情報を安全に扱うための環境。テクノプロジェクトが中心的価値を発揮する領域であり、単に一般的なAIツールを使えるようにすることとは異なる、技術的専門性を伴う基盤整備として位置づけます。
連携・補完
Layer 2 — 非医療情報系
ビジネス用 生成AI環境
非医療情報・非個人情報領域で公式に利用できる複数アカウント対応のビジネス向け生成AI環境。会議録作成や文書草案に留まらず、業務棚卸し・見える化・構造化・再構築案作成・DX介入後のあるべき姿の設計といった上流の業務改善プロセスに活用します。

二層構造にすることで、医療情報領域を射程に入れつつ、同時に現実的なスピードで導入できる領域を確保し、教育・定着を進めやすくなります。

```
03

教育パッケージと内製化の方針

教育パッケージの具体的内容は現時点では決めきれないとしても、事業の中に教育を明確に組み込み、基盤整備と並行して整えていく方向性については、この段階で共有・合意しておきたい事項です。

```

内製化は重要だが、すべてを100％内製化することを目指すわけではない。
基盤整備と教育によって病院側で担える範囲を広げつつ、より高度なアプリケーション開発や専門性の高いシステム構築は、引き続きエンジニアや外部専門家と連携して進めるべき領域として整理する。

この整理により、「どこまでを内製化し、どこからを専門開発として外部と協働するのか」という線引きが明確になり、持続可能なDX推進モデルとして提案の説得力が増すと考えています。

```
04

時間制約を踏まえた現実的アプローチ

本来であれば業務棚卸しから丁寧に入ることが理想ですが、提案書提出までの時間が極めて限られているのが現実です。

そこで、本来の方法論は提案内で示しつつも、現時点では事前の現地訪問やヒアリングを通じて把握している上五島病院側の課題群を起点に進める構成とします。すでに病院側から複数の重点課題が示されている以上、初年度はそれらを中心に重点ユースケースを設定し、限られた期間内でも実証として成立する設計に落とし込んでいく方が、採択可能性・実行可能性の両面で望ましいと判断しています。

05

高度アプリケーションの位置づけ ── SafeGENIE

SafeGENIEのような医療安全関連アプリケーションの試験的導入は、今回の実証において重要な位置づけを持ちます。

これは単なるツール紹介ではなく、「基盤の上に乗る高度アプリケーション」の代表例として位置づけられます。つまり、内製化可能な領域と、専門開発として外部と協働すべき領域を明示する——多層的な実証モデルの一部として語ることができる構成です。

★

提案全体の構造 ── 5つの柱

以上を踏まえ、今回の提案は以下の5つの柱で全体を構成します。

```
🔍
Pillar ①
現場課題の整理と業務再構築支援
ヒアリング・業務棚卸しから、課題の構造化と優先順位付け、そして再構築案の提示と病院の意思決定に基づく改善実行までを一貫して支援する。
🏗️
Pillar ②
医療情報系・非医療情報系の二層AI基盤整備
テクノプロジェクトの技術的専門性を核とした医療情報対応環境と、即効性のあるビジネス用生成AI環境を並行して構築する。
🎓
Pillar ③
職員・コアメンバーへの教育による内製化支援
教育パッケージを事業に組み込み、基盤整備と並行して自施設での改善継続能力を育成する。
⚡
Pillar ④
基盤上での重点ユースケース実証
上五島病院から示された重点課題を起点に、限られた期間でも成果を出せるユースケースを設定し、実証として成立させる。
🚀
Pillar ⑤
高度アプリケーション導入を含む多段階実証
SafeGENIE等を「基盤の上に乗る専門アプリ」として試験導入し、内製化領域と専門開発領域の線引きを明示する多層モデルを示す。
```
