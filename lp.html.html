Hospital AX / Generative AI

生成AIを、
病院の実装力に。

myGEMSは、医療機関の業務課題を整理し、生成AIを活用した改善設計・PoC・職員教育・内製化までを伴走する支援サービスです。

AIツールを入れて終わりにするのではなく、現場で使われ、院内に知見が残り、改善が続く仕組みをつくります。まずは「どこから変えるべきか」を一緒に明確にします。

Client Pain

そのDX、現場で止まっていませんか。

多くの医療機関では、AIやシステム導入の前に、業務の詰まり・役割の曖昧さ・推進体制の不足がボトルネックになっています。myGEMSは、現場の声を「ただの不満」で終わらせず、改善できる課題へ翻訳します。

Workload

現場負荷が高いのに、原因が見えない

二度入力、書類作成、確認作業、電話対応など、忙しさの原因が業務構造として整理されていない状態です。

AI Readiness

生成AIを使いたいが、安全性が不安

医療情報を扱う以上、職員任せの利用では不十分です。ルール、権限、教育、ログ、運用体制が必要です。

Governance

DX推進が担当者任せになっている

誰が意思決定し、誰が現場を巻き込み、どこまでを今期実施するかが曖昧だと、改善は継続しません。

Internalization

導入しても、知見が院内に残らない

外部に任せきりの改善では、運用後の微修正、他部署展開、次のテーマ設定で再び止まりやすくなります。

Philosophy

AI導入ではなく、改善が続く構造をつくる。

myGEMSの支援は、単なるツール紹介や研修ではありません。現場の困りごとを起点に、業務の詰まりを構造化し、改善案を設計し、小さく試し、院内で再現できる型として残すことを重視します。

01

主語は「技術」ではなく「現場」

何を使うかより、何に困っているかから始めます。

02

主眼は「デジタル化」ではなく「変革」

単純な置き換えではなく、業務手順や役割を見直します。

03

技術選択は目的に従わせる

生成AIを中心に、課題に応じて最適な方法を選びます。

04

伴走の目的は「依存」ではなく「自走」

院内に考え方と再現可能な型を残します。

地域や病院が、自律的に学習し、改善し続ける状態へ。

生成AIは目的ではなく、知識アクセス、構造化、文書化、教育を支える基盤です。

myGEMS Approach

一般的なDXツール導入とは一線を画す、5つの特徴。

このセクションは、丸い一文字アイコンを廃止し、クライアントが意味を直感的に理解できるラベル設計に変更しています。

Field First

現場起点

ツール導入ありきではなく、現場の困りごと、負荷、属人化、制約条件から出発します。機能ではなくペインポイントから入ります。

Structure First

構造化

不満を単発の事象として扱わず、業務プロセスや情報流通の構造的な問題として捉え直します。

AI Implementation

AX実装

生成AIを単なる便利ツールではなく、文書化・試作・改善設計を前に進める中核的な推進力として位置づけます。

Build In-House

内製化前提

外部ベンダーへの依存ではなく、院内人材が使いこなし、育て続ける自走体制をゴールにします。

Fit to Context

個別最適実装

汎用パッケージの押し付けではなく、地域性、施設規模、人材状況、既存システムに合わせた現実解へ落とします。

Project Boundary

本線・教育・後続展開を切り分ける、3層構造。

「何をどこまでやるか」を曖昧にしない。プロジェクト本線、並行する教育・内製化支援、終了後の定着化・横展開を分けて設計します。

Phase 0–7

机上の設計で終わらせず、現場運用まで落とし込む。

課題の探索から実証・評価まで、各フェーズで目的と成果物を定義し、意思決定の曖昧さを減らします。

Phase 0

前提条件整理・初期合意形成

病院種別、規模、システム環境、人員構成、推進余力を把握し、変革を受け止める体制を仮整理します。

Phase 1

情報収集体制設計と業務可視化

対象部署・キーマン選定、ヒアリング、As-Isフロー、例外処理、属人化、ボトルネックを可視化します。

Phase 2

課題定義・構造分析・優先順位付け

表層的な不満と本質原因を分け、今期対象・次期対象を整理します。

Phase 3

To-Be設計図作成

改善後の業務フロー、人とAIの役割分担、段階導入案、リスク、教育計画を描きます。

Phase 4

PoC / 実行計画策定

PoCテーマ、範囲、期間、体制、KPI、成功条件・中止条件を実行計画へ落とします。

Phase 5

経営判断・実行可否決定

投資対効果、リスク、実現性を経営層と現場で確認し、実行範囲と責任者を確定します。

Phase 6

PoC実施

現場運用、定例レビュー、プロンプト・フロー修正、ログ確認を短周期で回します。

Phase 7

評価・プロジェクト終了判定

定量成果、現場負荷、安全性、実用性、費用対効果、内製化可能性を評価します。

Secure AI Foundation

生成AIを、安全に使える基盤まで設計する。

医療機関で生成AIを活用するには、単なるプロンプト研修だけでは不十分です。セキュア環境、データガバナンス、権限管理、監査ログ、運用ルール、教育体系を含む「使える基盤」が必要です。
